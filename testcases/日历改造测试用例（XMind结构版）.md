## +产品详情

### +头部起价与报名信息

#### 用例01：起价规则适用品类与展示逻辑

##### 测试步骤
1. 准备以下适用产品：一级品类=亲子游的预订产品且库存类型=人数；一级品类=营地教育的预订产品且库存类型=人数；二级品类=亲子活动的预订产品且库存类型=人数。
2. 在亲子游、亲子活动产品中配置多个可见可售团期，其中至少一个团期同时有1成人价和1儿童价；在营地教育产品中配置含1儿童价的可见可售场次。
3. 分别进入上述产品详情页，查看头部起价展示。
4. 调整最低价场次后刷新页面，再次查看起价变化。

##### 预期结果
1. 仅对“一级品类=亲子游/营地教育”或“二级品类=亲子活动”，且同时满足“产品类型=预订产品”“库存类型=人数”的产品应用本次起价改造。
2. 亲子游、亲子活动默认按“同时含1成人+1儿童价格”的场次计算，展示成人起价与儿童起价，单位分别为“起/成人”“起/儿童”。
3. 当成人价与儿童价相同，合并展示一个价格，单位显示“起/人”。
4. 营地教育默认找所有可见可售且含1儿童价格的场次，展示儿童起价，单位为“起/人”。
5. 最低价场次发生变化后，头部起价跟随更新，同价时取团期开始日期更早的场次。

#### 用例02：起价降级逻辑与非适用品类边界

##### 测试步骤
1. 准备一级品类=亲子游的预订产品且库存类型=人数，移除所有“同时含1成人+1儿童”的场次，仅保留含1成人价场次。
2. 准备二级品类=亲子活动的预订产品且库存类型=人数，重复上述数据处理并进入详情查看。
3. 准备一级品类=营地教育的预订产品且库存类型=人数，仅保留含1儿童价场次，再移除1儿童价仅保留其他价格类型场次分别查看。
4. 准备不满足条件的对照产品，如非预订产品、库存类型非人数、或非上述品类产品，进入详情查看起价。

##### 预期结果
1. 亲子游、亲子活动找不到1成人+1儿童组合时，降级展示成人最低价，单位“起/人”。
2. 营地教育优先展示儿童最低价；若找不到1儿童价格，则按兜底逻辑展示可见可售最低价，不展示单位。
3. 目标品类都找不到目标价格时，按现有逻辑展示所有可见可售场次最低起价，不管价格类型且不展示单位。
4. 不满足“目标品类 + 预订产品 + 库存类型=人数”条件的产品，不套用本次起价新逻辑，保持原有展示逻辑。

#### 用例03：报名人数文案与样式改造

##### 测试步骤
1. 准备报名人数分别为普通数值、超过1w、千分位为0的数据。
2. 分别在“有横幅/无横幅”“同时展示成人+儿童价且位数较长”的场景进入详情页。
3. 观察报名人数文案和排版。

##### 预期结果
1. 文案为“XXX人已报名”，不再展示“累计已有”。
2. 超过1w时展示到千分位，如“1.4w+人已报名”，千分位为0时展示“1w+人已报名”。
3. 在无横幅且价格位数长的特定场景，报名人数按需求切为上下两行展示。

#### 用例04：价格说明入口热区

##### 测试步骤
1. 准备“有调价横幅”和“无横幅”两种产品。
2. 在有横幅场景点击整个横幅区域。
3. 在无横幅场景点击包含起价的第一行区域。
4. 准备“无问号解释”场景，点击原调价优惠金额入口。

##### 预期结果
1. 有横幅时，整个横幅区为可点击热区并打开价格说明弹窗。
2. 无横幅时，起价所在第一行区域为可点击热区并打开价格说明弹窗。
3. 无问号解释时，原优惠金额入口可正常打开价格说明弹窗。

#### 用例05：头部到底部价格取值一致性

##### 测试步骤
1. 准备存在拼单立减的产品，进入详情页记录头部最低价与优惠前原价。
2. 观察底部按钮价格展示。
3. 再准备无拼单立减的产品重复验证。

##### 预期结果
1. 底部按钮价格取值为头部最低价。
2. 有拼单立减时，底部展示两个价格（最低价 + 对应拼单优惠前原价）。
3. 无拼单立减时，仅展示一个价格。

### +价格说明弹窗

#### 用例06：弹窗信息结构与文案

##### 测试步骤
1. 在产品详情点击价格说明入口打开弹窗。
2. 分别查看常规售价、优惠项、当前起价、起价说明区域。
3. 准备起价为组合价（1成人1儿童）与非组合价两种场景对比查看。

##### 预期结果
1. 常规售价展示起价对应场次的原价。
2. 优惠项展示调价和拼单立减，调价文案含规则与截止/剩余信息。
3. 当前起价与详情页保持一致，存在拼单返现时展示“拼成再返”文案。
4. 组合价场景下优惠单位与金额计算按组合人数规则展示，不误加“/人”。
5. 起价说明按需求展示高亮线路团期、儿童说明（有则展示）和固定说明文案。

#### 用例07：弹窗底部按钮分支（可售/预售/不可售）

##### 测试步骤
1. 准备产品可售场景，打开价格说明弹窗并点击底部按钮。
2. 准备产品预售场景重复操作。
3. 准备产品不可售场景重复操作。

##### 预期结果
1. 可售时按钮为“计算我的出行价格”，点击后关闭当前弹窗并打开日历弹窗。
2. 预售时按钮为“开售提醒我”，点击逻辑与详情底部开售提醒一致。
3. 不可售时按钮为“我知道了”，点击仅关闭弹窗。

### +团期卡片（详情页）

#### 用例08：库存状态文案展示

##### 测试步骤
1. 准备团期库存分别为<10、>=10、不可售可候补、不可售不可候补四类数据。
2. 进入详情页查看团期卡片状态文案。
3. 准备老版本客户端（9.0.0及之前）对比查看。

##### 预期结果
1. 可售且库存<10显示“仅余XX人/间房/车”。
2. 可售且库存>=10显示“报名中”。
3. 不可售可候补显示“余位通知”。
4. 不可售不可候补不显示库存占位字段，仅保留“已满员”。
5. 移除“可候补/余位紧张”旧标签（老版本按兼容策略保持原样）。

#### 用例09：月份Tab和“查看更多团期”改造

##### 测试步骤
1. 准备跨多个月份团期数据，进入详情页查看月份Tab。
2. 准备超过6个月有团期的数据，查看月份Tab数量。
3. 点击“查看更多团期”按钮。

##### 预期结果
1. 月份Tab按“最多6个有团期月份”展示，不是“当前日期+6个月”。
2. 按钮文案为“查看更多团期”。
3. 点击后打开全部出行日期弹窗。

## +日历弹窗

### +预订无多场次（亲子游一级品类/多日营二级品类）

#### 用例10：进入日历弹窗默认态与套餐区展示

##### 测试步骤
1. 从可售产品详情进入日历弹窗，分别准备“一级品类=亲子游”“二级品类=多日营”的可售预订产品，并覆盖“多线路”“单线路”场景。
2. 观察线路/套餐区文案与高亮状态。
3. 分别准备“有团期无套餐”和“无团期有套餐”产品，观察套餐区展示方式。

##### 预期结果
1. 一级品类=亲子游、二级品类=多日营的产品，用户端不展示场次维度的数据和选择。
2. 多线路产品展示已选线路名称+天数及套餐选择，已选套餐高亮。
3. 单线路产品展示固定标题+套餐选择，已选套餐高亮。
4. 有团期无套餐场景展示全部套餐；无团期有套餐场景仅展示已选套餐（按补充规则）。

#### 用例11：月份与团期默认选中逻辑

##### 测试步骤
1. 进入日历弹窗且携带日期参数。
2. 进入日历弹窗但不携带日期参数。
3. 准备只有一个月份与多个月份两种数据。
4. 准备同一开始-结束日期下多个场次数据，验证展示取值。

##### 预期结果
1. 带日期参数时自动定位并选中对应月份与团期。
2. 不带日期参数时默认定位第一个月份，不默认选中具体团期。
3. 仅一个月份时不展示月份Tab。
4. 同开始-结束日期默认取可售最新创建场次；开始时间相同结束不同按两个团期展示。

#### 用例12：出行人数、年龄标识与人数携带

##### 测试步骤
1. 选择团期后查看成人/儿童人数区，验证不同人数组合展示。
2. 准备一级品类=亲子游的产品且存在单儿童规则，查看“不占床”标识位置。
3. 先在团期A选择人数，再切换到团期B观察人数是否携带。
4. 构造切换后年龄带不匹配场景，继续切换团期。

##### 预期结果
1. 人数文案按规则展示（如1成人&1儿童、1成人&2儿童等）。
2. 一级品类=亲子游场景按规则打“不占床”标识，多年龄规则时默认标在第一个单儿童后。
3. 切换团期默认携带已选人数。
4. 若新团期年龄带不匹配，不携带并提示“所选日期出行人年龄限定发生变化，请重新选择”。

#### 用例13：房间数/车辆数补差交互

##### 测试步骤
1. 选择会触发房差的团期与人数，观察房间数区域文案。
2. 点击房间数说明交互区。
3. 选择会触发车差的团期与人数，观察车辆数区域文案。
4. 点击车辆说明交互区。

##### 预期结果
1. 仅在需要补差时展示房差/车差说明文案与补差金额。
2. 点击房间数区域打开房差说明弹窗。
3. 点击车辆数区域触发咨询专员获客路径（按端展示落地形态）。

#### 用例14：底部栏与下一步按钮（可售场次）

##### 测试步骤
1. 选中可售团期但不选人数，观察底部价格和副文案，点击“下一步”。
2. 继续选择完整的出行人数，观察底部价格、优惠标识和明细入口。
3. 切换套餐，观察底部“已选提醒”展示。

##### 预期结果
1. 未选人数时价格显示“-”，副文案提示请选择必要项，点击下一步触发对应toast。
2. 选完人数后按规则计算价格并展示明细入口；有优惠时显示优惠标识。
3. 已选提醒展示“团期开始日期+套餐名称”，最多一行，超出省略。

#### 用例15：可候补场次底部双按钮

##### 测试步骤
1. 选中不可售可候补场次，观察底部按钮区。
2. 点击“添加专员”按钮。
3. 点击“余位通知”按钮。
4. 准备无咨询群配置产品重复验证。

##### 预期结果
1. 底部展示“添加专员 + 余位通知”双按钮。
2. 添加专员走获客链路（小程序展示二维码弹窗）。
3. 余位通知打开登记弹窗。
4. 无咨询群配置时按需求仅保留可用按钮，不展示无效咨询入口。

#### 用例16：优惠金额计算与展示

##### 测试步骤
1. 准备同时存在调价优惠、拼单立减、会员价优惠的场次并选择人数。
2. 观察日历页“已减优惠/拼成再返”展示。
3. 调整人数后再次观察优惠金额变化。

##### 预期结果
1. 已减优惠=调价优惠+拼单立减+会员价优惠合计，随人数实时联动。
2. 拼单后返优惠金额按规则计算并随人数联动。
3. 优惠展示与金额明细弹窗保持一致。

### +全部日期弹窗

#### 用例17：弹窗样式、跳转与关闭回退

##### 测试步骤
1. 在日历页点击“查看全部日期”进入弹窗。
2. 观察弹窗出现动效。
3. 点击顶部月份Tab，观察下方日期区定位。
4. 点击关闭按钮返回。

##### 预期结果
1. 弹窗从底部弹出，不是直接出现。
2. 顶部展示所有月份和对应起价。
3. 点击月份Tab可快捷滚动到对应月份并切换高亮。
4. 关闭后回到上层日历与人数弹窗。

#### 用例18：日期状态渲染与可选性

##### 测试步骤
1. 准备节假日、今天、历史日期、无场次、未成团、已满员、可售、可候补等多状态数据。
2. 打开全部日期弹窗，逐类观察日期样式、文案和可点击性。
3. 分别点击可选和不可选日期。

##### 预期结果
1. 节假日当天显示节假日名，其他节假日显示“休”；今天文案优先级低于节假日规则。
2. 无场次/未成团日期灰态不可选；已满员/已关闭/已截止日期高亮“已满员”不可选。
3. 可售日期展示起价与对应状态文案（仅余XX/报名中/有余位通知我/已成团优先级）。
4. 可选日期可点击带回日历页，不可选日期点击无选中并保持禁用表现。

#### 用例19：同日多场次取值规则（全部日期）

##### 测试步骤
1. 在一级品类=亲子游、二级品类=多日营产品中分别准备同一日期多场次数据（可售、成团、不同结束时间）。
2. 打开全部日期弹窗观察该日期展示。
3. 在除一级品类=亲子游、二级品类=多日营外的其他品类产品重复验证。

##### 预期结果
1. 一级品类=亲子游、二级品类=多日营默认取“可售且结束时间最早”场次作为日期展示依据。
2. 除一级品类=亲子游、二级品类=多日营外的其他品类，按补充规则取“可售且成团且开始时间最早”场次。
3. 日期标签、价格、状态文案均与选中的默认场次一致。

### +附属弹窗

#### 用例20：余位短信提醒弹窗

##### 测试步骤
1. 在候补场景点击“余位通知”打开登记弹窗。
2. 验证线路+套餐名称、日期、人数展示；分别测试外层已选人数与未选人数场景。
3. 输入非法手机号点击登记，再输入合法手机号提交。

##### 预期结果
1. 标题和信息区展示已选线路套餐（支持换行全展示）、日期区间与可编辑人数。
2. 手机号非法时toast提示“请填写合法的手机号”。
3. 登记成功后关闭弹窗并回到日历弹窗，toast提示“登记成功，请关注短信提醒”。

#### 用例21：房差说明与金额明细弹窗

##### 测试步骤
1. 在日历页触发房差说明弹窗。
2. 点击弹窗内“咨询专员”。
3. 在日历页点击“金额明细”入口打开价格明细弹窗。

##### 预期结果
1. 房差弹窗展示固定图片+文案，信息完整可读。
2. 咨询专员按钮走获客链路，与其他咨询入口一致。
3. 金额明细弹窗展示基础商品与优惠明细（成人/儿童/房差/车差/拼单立减/早鸟立减/会员价等）。

### +预订有多场次（除亲子游一级品类/多日营二级品类外的其他品类）

#### 用例22：多场次选择与场次携带

##### 测试步骤
1. 进入除一级品类=亲子游、二级品类=多日营外的其他品类产品日历弹窗，检查是否展示出行时间模块。
2. 准备单场次和多场次日期，观察默认选中逻辑。
3. 先选择日期A的场次1，再切换到日期B，验证同起止时间场次携带规则。
4. 点击不可售不可候补场次。

##### 预期结果
1. 除一级品类=亲子游、二级品类=多日营外的其他品类展示场次维度与选择；一级品类=亲子游、二级品类=多日营不展示该模块。
2. 单场次默认选中，多场次不默认选中，支持横向滑动查看。
